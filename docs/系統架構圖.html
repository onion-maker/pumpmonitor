2. 資料流 — API → Store → UI

sequenceDiagram
    participant API1 as 🏛️ heovcenter API
    participant API2 as 🏛️ heovcenter2 API
    participant FETCH as api/pumpStation.ts
    participant HOOK as usePumpData.ts
    participant STORE as Zustand Store
    participant CARD as StationCard
    participant BG as PumpMonitorService

    loop 每 30 秒
        HOOK->>FETCH: fetchAllStations()
        FETCH->>API1: GET /GetLastestAutoPumpPGInfo
        FETCH->>API2: GET /GetLastestAutoPumpPGInfo (備援)
        API1-->>FETCH: { d: RawStationData[] }
        API2-->>FETCH: { d: RawStationData[] }
        FETCH->>FETCH: mergeStations() 各站取 rectime 最新
        FETCH->>FETCH: normalizeStation() 正規化
        FETCH-->>HOOK: PumpStationData[]
        HOOK->>STORE: setStationData(data)
        HOOK->>STORE: recordLevelOut() 潮汐 buffer
        alt 距上次 > 10分鐘
            HOOK->>STORE: checkTideAlarm()
        end
        HOOK->>STORE: checkAlarm(data)
        STORE->>CARD: stationData + alarmStates
    end

    loop 背景間隔 (預設 120s)
        BG->>API1: HttpURLConnection GET
        BG->>API2: HttpURLConnection GET (備援)
        BG->>BG: merge + checkAlarm + tide check
        BG->>BG: sendAlarmNotification() (如有警報)
    end
